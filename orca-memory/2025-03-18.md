### Initial Findings

**Project Identity:** Orca is a full-stack, AI-powered, Web3-native Supply Chain Management (SCM) platform with real-world integrations and modern UI.

**Key Features Confirmed:**
- Shipment tracking smart contract (Solidity)
- NFT passports for goods
- IPFS metadata storage
- AI-powered shipment delay prediction (Isolation Forest model)
- Inventory Dashboard with real-time stock view
- Predictive analytics (run-out forecasting)
- Automated ordering (reorder threshold triggers)
- Supplier analytics (processing time)
- Purchase Order Management with approval workflow
- Supplier Suggestion Engine (backend implemented)
- Weather/GPS oracles
- Computer vision integration (Roboflow)
- DAO voting
- Chainlink oracle support
- User authentication (OAuth2 + JWT)
- API rate limiting
- PostgreSQL for persistent storage
- Redis for caching/rate limiting

**Tech Stack Identified:**
- Backend: FastAPI, Python 3.10, SQLAlchemy, PostgreSQL, Redis
- Frontend: Next.js 13.4, React 18, Material-UI (MUI), Framer Motion, Ethers.js
- Blockchain: Solidity ^0.8.0, Hardhat, Ganache (local), Web3.py
- Storage: IPFS (Pinata integration)
- AI/ML: scikit-learn, pandas, numpy, Roboflow (computer vision)
- Deployment: Docker, Docker Compose
- External APIs: OpenWeatherMap, Roboflow, Chainlink

**Architecture Pattern:**
- Monorepo structure with frontend/, backend/, ai/, web3/ directories
- Backend: FastAPI app with DAO pattern for database operations
- Frontend: Next.js with API routes proxied to backend
- Web3: Separate Hardhat project with deployment scripts
- AI: Simple ML models in backend/ai/ and top-level ai/ directories

**Database Schema:**
- inventory (PostgreSQL) - product info, quantities, thresholds, supplier data
- shipments - shipment tracking with IPFS hashes and blockchain tx info
- purchase_orders - PO lifecycle management
- users - authentication/authorization

**Security Observations:**
- Private keys stored in environment variables (SHIPMENT_SENDER_PK, NFT_SENDER_PK) - needs secure management
- JWT-based authentication implemented
- Rate limiting via fastapi-limiter + Redis
- Password hashing with bcrypt via passlib

**Known Issues from analysis.md:**
- Code duplication in frontend components (error handling, loading states)
- API call patterns duplicated
- Inconsistent error handling for Web3 interactions
- AI models basic (though enhanced from rules-based)
- No notification system
- Limited visualization
- Supplier suggestion engine frontend integration pending
- PO workflow incomplete (basic approval only)
